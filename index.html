Application Architecture in Front-end

by Nastassia Krasutskaya

Basic rules

  • Easy expansion of the application functionality
  • Simple changes to existing functionality
  • Fast onboarding of new developers to the project
  • Clear and transparent code
  • Convenient and fast navigation

Popular architecture patterns

  • Model-View-Controller (MVC)
  • Clear architecture
  • Uni-directional data flow (Flux)

Model-View-Controller

Structure

  • Model (data and business logic of the application)
  • View (presentation and user interface)
  • Controller (intermediary between the model and the view)
